  <files>iosApp/iosApp/BookFeedView.swift</files>
  <action>
**Root Cause:** `.ignoresSafeArea()` is applied to the child `BookCoverCard` inside a ScrollView, but the ScrollView container itself respects safe areas, which blocks the child from extending into them.

**Issue:** "no - book cover is not extending into top and bottom safe areas"

**Expected:** Book covers display full-screen extending into safe areas.

**Fix:**
1. Apply `.ignoresSafeArea()` to the ScrollView container itself (not just the child views)
2. This allows the ScrollView content to extend into the top and bottom safe areas
3. The book cover images will then properly fill the full screen including the notch/home indicator areas

**Implementation:**
- Find the ScrollView in BookFeedView
- Add `.ignoresSafeArea()` modifier to the ScrollView
- Keep or remove the child `.ignoresSafeArea()` on BookCoverCard (both approaches should work, but applying to parent is the key fix)
  </action>
